# Antigravity Global Rules

## 언어
- 모든 응답은 한국어로 작성
- 사고 과정, PRD, 중간 산출물, 에러 분석 모두 한국어
- 변수명/함수명은 영어 유지 (코드 컨벤션)

## 응답 방식
- 작업 시작 전 계획과 코드 변경 이유는 핵심 위주로 간결하게(1~2줄 내외) 설명하고 즉시 작업을 진행
- 새로운 기능이나 복잡한 로직(계산 등)을 구현할 때는, 실제 코드 작성 전 정상 작동과 예외 상황(Edge Cases)을 검증할 수 있는 간단한 테스트 기준이나 확인 스크립트를 먼저 제시
- 에러 발생 시 원인과 해결책을 명확히 제시
- 불확실한 부분은 임의로 추측하여 코딩하지 말고 명시적으로 질문하여 확인

## 구현 순서 원칙 (단계별 검증)
- 전체를 한 번에 연결하지 않고 아래 순서로 단계별 검증 후 진행
  1. **백엔드 단독 검증**: API 엔드포인트 단독 실행 → curl 또는 Swagger로 응답 확인
  2. **프론트 단독 검증**: UI 렌더링 및 상태 변화 확인 → mock 데이터로 화면 동작 확인
  3. **연결 검증**: 프론트 ↔ 백엔드 실제 연동 테스트
- 각 단계에서 정상 동작 확인 전까지 다음 단계로 넘어가지 않음
- 에러 발생 시 전체 코드를 수정하기 전에 어느 단계에서 문제가 생겼는지 먼저 특정

## 환경 확인 원칙
- 코드 실행 전 반드시 가상환경 활성화 여부 확인 (`which python` / `where python`)
- 새 패키지 설치 후 `pip freeze > requirements.txt` 즉시 갱신
- import 에러 발생 시 코드 수정 전 환경 문제 여부를 먼저 진단

## 코딩 스타일 및 구조
- 화면 표시(UI), 데이터 저장/불러오기, 핵심 비즈니스 로직을 명확히 구분하여 별도의 파일로 분리 (KISS 원칙 준수, 불필요한 파일 쪼개기 지양)
- Python: PEP 8 준수
- 핵심 비즈니스 로직이나 복잡한 함수/클래스에 한하여 한국어 docstring 및 주석 작성
- 매직 넘버 사용을 금지하고 상수로 분리

## 안전 규칙
- 기존 파일의 삭제 및 대규모 덮어쓰기 전에는 반드시 확인 요청
- .env 파일 및 API 키, 비밀번호 등 민감 정보는 절대 코드에 하드코딩 금지
